Top row: The web application itself — user interacts via browser, Flask handles requests, SQLite stores data.
Bottom row: The test pipeline — Behave reads .feature files and triggers Selenium, which drives a headless browser against the running Flask app.

What Each Component Does

⚙
Flask (app.py) — The web server. Serves 3 routes: /login for authentication, /create-test for the test creation form, and /tests to list saved tests. Renders HTML templates and handles form submissions.
🗃
SQLite (db.py) — The database. Stores test scenarios in a test_scenarios table with columns: name, application_url, steps_raw, expected_outcome, and status (defaults to "Not Run"). Reset before each test run for isolation.
⚙
Behave — The BDD test runner. Reads .feature files (Gherkin syntax), matches each Given/When/Then step to a Python function in steps/, and reports pass/fail.
🌐
Selenium + Chromium — The browser automation. Opens a headless (invisible) Chrome browser, navigates to pages, fills in form fields with send_keys(), clicks buttons with click(), and reads page content with page_source to verify assertions.
📦
Docker — The container. Packages Python, Flask, Chromium, ChromeDriver, and all dependencies into one reproducible image. One command (docker-compose run) builds and runs everything — no local setup needed.
▶
environment.py — The glue. Starts Flask in a background thread, initializes Selenium, resets the database before each scenario, and tears everything down after tests complete.

Feature: Test Scenario Creation
  As a QA engineer, I want to create test scenarios in natural language

  Scenario: Create a simple web application test using natural language
    Given I am logged into the testing platform
    When I navigate to the "Create Test" page
    And I enter the test name "Checkout Flow Validation"
    And I enter the application URL "https://example-shop.com"
    And I provide the test steps in natural language:
      "1. Navigate to homepage 2. Click Products 3. Select first product
       4. Click Add to Cart 5. Verify cart shows 1 item"
    And I set expected outcome "Cart displays 1 item successfully"
    And I click "Save Test"
    Then I should see a confirmation message "Test scenario created successfully"
    And the test should appear in my test list with status "Not Run"

Feature: Intelligent Failure Diagnosis

Selenium: What Happens Under the Hood

The automated test does exactly what a human does — but in a headless browser

You (Manual)

1. Open browser, go to /login
2. Type email & password, click Login
3. Navigate to /create-test
4. Type "Checkout Flow Validation"
5. Type the application URL
6. Type test steps in the textarea
7. Type expected outcome
8. Click Save Test
9. See confirmation & test in list

Selenium (Automated)

1. driver.get("/login")
2. find_element(ID, "email").send_keys(...)
3. driver.get("/create-test")
4. find_element(ID, "test_name").send_keys(...)
5. find_element(ID, "application_url").send_keys(...)
6. find_element(ID, "steps_raw").send_keys(...)
7. find_element(ID, "expected_outcome").send_keys(...)
8. find_element(ID, "save-test-btn").click()
9. assert "Test scenario created" in page

Runs in headless Chromium inside Docker — no visible browser window needed
